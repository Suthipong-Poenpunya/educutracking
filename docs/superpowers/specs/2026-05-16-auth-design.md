# Auth System Design — ChulaEduTracker
**วันที่:** 2026-05-16
**สถานะ:** Approved

---

## ภาพรวม

แยก flow การ **เข้าสู่ระบบ** และ **สมัครใหม่** ออกจากกันอย่างชัดเจน โดยแสดงเป็น card เดียวที่ toggle state ใน browser โดยไม่ reload หน้า

ระบบ auth ใช้ **รหัสนิสิต + ชื่อ-นามสกุล (exact match, case-sensitive)** เป็นกุญแจยืนยันตัวตน ไม่มีการเพิ่ม column ใน Users sheet

## Logic การทำงาน

### Login Flow

1. ผู้ใช้กรอก `studentId` + `name`
2. เรียก `API.getUserByStudentId(studentId)`
3. ถ้า `res.success === false` → toast "ไม่พบรหัสนิสิตนี้ในระบบ"
4. ถ้า `res.data.display_name !== name` (exact, case-sensitive) → toast "ชื่อ-นามสกุลไม่ถูกต้อง"
5. ถ้าผ่าน → `currentUser = res.data` → `saveToStorage` → `showApp()`

### Register Flow

1. ผู้ใช้กรอกข้อมูลครบ
2. ตรวจ `studentId` ซ้ำก่อน: เรียก `API.getUserByStudentId(studentId)`
3. ถ้าพบแล้ว → toast "รหัสนิสิตนี้มีผู้ใช้อยู่แล้ว กรุณาเข้าสู่ระบบ"
4. ถ้าไม่พบ → เรียก `API.createUser(data)` → `showApp()`

### Toggle

- `switchAuthMode('login')` / `switchAuthMode('register')` — toggle `data-mode` attribute บน card
- ล้าง form + error ทุกครั้งที่ switch

## ไฟล์ที่แก้ไข

| ไฟล์ | การเปลี่ยนแปลง |
|------|----------------|
| `index.html` | แทนที่ `#register-screen` ด้วย auth card 2 state |
| `app.js` | เพิ่ม `handleLogin()`, `switchAuthMode()`, แก้ `handleRegister()` |
| `Code.gs` | ไม่ต้องแก้ (endpoint + schema ครบอยู่แล้ว) |
| `api.js` | ไม่ต้องแก้ (localGet case ครบอยู่แล้ว) |

## Error Messages

| กรณี | ข้อความ |
|------|---------|
| ไม่พบ studentId | "ไม่พบรหัสนิสิตนี้ในระบบ กรุณาสมัครใหม่" |
| ชื่อไม่ตรง | "ชื่อ-นามสกุลไม่ถูกต้อง กรุณาตรวจสอบและลองใหม่" |
| studentId ซ้ำตอนสมัคร | "รหัสนิสิตนี้มีผู้ใช้อยู่แล้ว กรุณาเข้าสู่ระบบ" |
| กรอกข้อมูลไม่ครบ | "กรุณากรอกข้อมูลให้ครบถ้วน" |

## ข้อจำกัด

- ไม่มี "ลืมรหัส" flow — ถ้าลืมชื่อที่กรอกไว้ ต้องติดต่อผู้ดูแลระบบ (by design)
- ไม่มี session expiry — ใช้ localStorage เหมือนเดิม
- Name matching เป็น exact + case-sensitive ตามที่ตกลง
